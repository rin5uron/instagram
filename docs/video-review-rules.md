# 動画振り返りルール

## 運用方法

各動画について、以下の2箇所で振り返りを記録する:

### 1. Issue（簡易振り返り）
- **タイトル**: `【ログ】001_焼きそば_振り返り`
- **ラベル**: `log`
- **内容**:
  - 撮影時間
  - 編集時間
  - 良かった点
  - 気になった点
  - 次回への改善案
  - 台本ファイルへのリンク（例: `scripts/001.md`）

### 2. 台本ファイル（詳細記録）
`scripts/XXX.md` に以下のセクションを追記:

```markdown
---

## 実際の構成（完成版）

（文字起こし内容を貼り付け）

## 振り返り

### 制作時間
- 撮影時間: X時間
- 編集時間: X時間
- 合計: X時間

### 良かった点
-
-

### 気になった点 / 改善点
-
-

### 次回への改善案
-
-
```

## Issue作成例

```bash
gh issue create --title "【ログ】001_焼きそば_振り返り" --body "$(cat <<'EOF'
<p align="right">2026-04-22</p>

## 制作時間
- 撮影: 3時間
- 編集: 1.5時間
- 合計: 4.5時間

## 良かった点
- 賞味期限切れ焼きそばのリアル感
- ChatGPT画面で具体的な質問を見せられた
- 「作る＋守る」のコピーが秀逸
- 麺はすでにほぐれていたのオチ

## 気になった点
- 買い出しシーンを入れた（削る予定だったが残した）
- 編集時間が目標より長かった

## 次回への改善案
- 固定テロップをテンプレート保存して時短
- 素材の命名ルール決めて、AirDrop時にリネーム
- 調理シーンの倍速を積極的に使う

## 台本ファイル
[scripts/001.md](https://github.com/rin5uron/instagram/blob/main/scripts/001.md)

EOF
)" --label log
```

## メリット

- **Issue**: 検索しやすい、一覧で比較できる
- **台本ファイル**: 詳細な構成記録、git履歴で変更追跡
- **リンク**: Issueから台本ファイルにすぐ飛べる
